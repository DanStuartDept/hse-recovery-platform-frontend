# Docusaurus Documentation Site — Design Spec

**Date:** 2026-04-09
**Status:** Draft
**Audience:** Frontend developers, DevOps/platform engineers

## Overview

Add an HSE-branded Docusaurus 3 documentation site to the monorepo at `apps/docs`. It serves as the central reference for the entire monorepo — architecture, packages, app guides, deployment, and site onboarding — targeting both frontend developers and DevOps engineers equally. Authored Markdown guides are supplemented by auto-generated API reference pages via TypeDoc.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Framework | Docusaurus 3.10.0 | First-class GitHub Pages support, MDX, sidebar navigation, search |
| Workspace integration | Full Turborepo member (`apps/docs`) | Consistent with monorepo conventions; builds, lints, typechecks via pipeline |
| Branding | Full HSE branding via CSS custom properties | Matches HSE service manual; uses design system tokens for colours, typography |
| API docs | `docusaurus-plugin-typedoc` (multi-instance) | Auto-generated from TypeScript source; stays in sync with code |
| Hosting | GitHub Pages | Free, simple, CI-deployed on push to `main` |
| Linting | Biome (extends `@repo/biome-config/base`) | Consistent with rest of monorepo |
| Type checking | TypeScript (extends `@repo/typescript-config/base`) | Consistent with rest of monorepo |
| Content authoring | Hand-written Markdown + Copilot agent | Full editorial control for guides; AI-assisted content generation |

## TypeDoc API Reference

### Plugin Stack

- `typedoc` — core TypeScript documentation generator
- `typedoc-plugin-markdown` — renders TypeDoc output as Markdown (required by the Docusaurus plugin)
- `docusaurus-plugin-typedoc` — integrates TypeDoc into the Docusaurus build pipeline

### Multi-Instance Configuration

Each documented package gets its own plugin instance in `docusaurus.config.ts`:

```ts
plugins: [
  ['docusaurus-plugin-typedoc', {
    id: 'api-client',
    entryPoints: ['../../packages/wagtail-cms-client/src/index.ts'],
    tsconfig: '../../packages/wagtail-cms-client/tsconfig.json',
    out: 'api/wagtail-api-client',
  }],
  ['docusaurus-plugin-typedoc', {
    id: 'cms-types',
    entryPoints: ['../../packages/wagtail-cms-types/src/index.ts'],
    tsconfig: '../../packages/wagtail-cms-types/tsconfig.json',
    out: 'api/wagtail-cms-types',
  }],
  ['docusaurus-plugin-typedoc', {
    id: 'cms-mapping',
    entryPoints: ['../../packages/wagtail-cms-mapping/src/index.ts'],
    tsconfig: '../../packages/wagtail-cms-mapping/tsconfig.json',
    out: 'api/wagtail-cms-mapping',
  }],
  ['docusaurus-plugin-typedoc', {
    id: 'app-config',
    entryPoints: ['../../packages/app-config/src/index.ts'],
    tsconfig: '../../packages/app-config/tsconfig.json',
    out: 'api/app-config',
  }],
  ['docusaurus-plugin-typedoc', {
    id: 'i18n',
    entryPoints: ['../../packages/i18n/src/index.ts'],
    tsconfig: '../../packages/i18n/tsconfig.json',
    out: 'api/i18n',
  }],
]
```

### Sidebar Integration

Each TypeDoc instance generates a `typedoc-sidebar.ts` file. These are imported and merged into the main `sidebars.ts` under an "API Reference" top-level category, with sub-categories per package.

### Build Order

Turborepo ensures `^build` (all dependent packages) completes before `docs#build` runs, so TypeDoc has access to compiled declarations.

## GitHub Pages Deployment

### Workflow: `.github/workflows/docs.yml`

**Trigger:** Push to `main` with changes in:
- `apps/docs/**`
- `packages/*/src/**` (API reference source)
- `.github/workflows/docs.yml`

**Steps:**
1. Use existing `.github/actions/setup` action (Node 24, pnpm, npm token)
2. `pnpm install`
3. `turbo run build --filter=docs` (builds dependent packages, then Docusaurus)
4. Deploy `apps/docs/build/` via `actions/deploy-pages`

### Docusaurus Config Values

```ts
url: 'https://dept.github.io',
baseUrl: '/hse-multisite-frontend/',
organizationName: 'dept',
projectName: 'hse-multisite-frontend',
trailingSlash: false,
```

## Turborepo Integration

### Scripts (`apps/docs/package.json`)

| Script | Command | Purpose |
|---|---|---|
| `dev` | `docusaurus start` | Dev server with hot reload |
| `build` | `docusaurus build` | Production static site |
| `serve` | `docusaurus serve` | Preview production build |
| `typecheck` | `tsc --noEmit` | Type checking |
| `lint` | `biome check --write` | Linting + auto-fix |

### Pipeline

- `docs#build` — depends on `^build`, outputs `build/**`
- `docs#dev` — persistent, no cache
- `docs#lint` and `docs#typecheck` — use default pipeline config

### .dockerignore Update

Add `apps/docs` to the exclusion list so the docs site is never included in app Docker build contexts.

## Dependencies

### Production

| Package | Version | Purpose |
|---|---|---|
| `@docusaurus/core` | `3.10.0` | Docusaurus core |
| `@docusaurus/preset-classic` | `3.10.0` | Default theme + plugins |
| `react` | `^18.0.0` | Docusaurus 3.x requires React 18 (not the monorepo's React 19 catalog version) |
| `react-dom` | `^18.0.0` | Must match React version |

> **Note:** Docusaurus 3.x is not compatible with React 19. The `apps/docs` workspace must pin its own React 18 versions rather than using `catalog:`. This is safe — pnpm workspaces allow different versions in different packages.

### Development

| Package | Version | Purpose |
|---|---|---|
| `typedoc` | latest | TypeScript documentation generator |
| `typedoc-plugin-markdown` | latest | Markdown output for TypeDoc |
| `docusaurus-plugin-typedoc` | latest | Docusaurus TypeDoc integration |
| `@repo/biome-config` | `workspace:*` | Shared Biome rules |
| `@repo/typescript-config` | `workspace:*` | Shared tsconfig base |

## Content Authoring

All guide content (non-API pages) is hand-authored Markdown. Content will be written by Copilot agent, drawing from:

- Existing package READMEs
- CLAUDE.md project documentation
- Source code and TypeScript types
- Existing docs in `docs/` directory
- Docker and CI/CD configuration files

The "Onboarding Sites" section will be populated once the site templating workflow is established.

## Out of Scope

- Blog / changelog (disabled initially, can be enabled later)
- Search (Docusaurus has built-in search that works out of the box for smaller sites; Algolia can be added later if needed)
- Versioned docs (single version for now; versioning can be added when the project stabilises)
- Storybook for component playground (can be added separately if component library grows)
