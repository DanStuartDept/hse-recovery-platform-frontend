# Git Hooks Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enforce Conventional Commits and run lint/typecheck/test locally on every commit using Husky + lint-staged.

**Architecture:** Husky manages two git hooks (`commit-msg`, `pre-commit`). lint-staged runs Biome on staged `.ts`/`.tsx` files. Turbo runs typecheck and test across affected packages. All hooks are committed to the repo and auto-installed via the `prepare` script.

**Tech Stack:** Husky, lint-staged, @commitlint/cli, @biomejs/biome, Turborepo

### Task 3: Configure the commit-msg hook

**Files:**
- Create: `.husky/commit-msg`

- [ ] **Step 1: Create the commit-msg hook**

Write `.husky/commit-msg` with this content:

```sh
pnpm commitlint --edit $1
```

No shebang needed — Husky v9+ handles this automatically.

- [ ] **Step 2: Verify commitlint rejects a bad message**

Run:

```bash
echo "bad message" | pnpm commitlint
```

Expected: FAIL — commitlint rejects the message (not Conventional Commits format). Output will contain something like `subject may not be empty` or `type may not be empty`.

- [ ] **Step 3: Verify commitlint accepts a good message**

Run:

```bash
echo "chore(configs): test commitlint" | pnpm commitlint
```

Expected: PASS — no errors.

- [ ] **Step 4: Commit the commit-msg hook**

```bash
git add .husky/commit-msg
git commit -m "chore(configs): add commitlint commit-msg hook"
```

### Task 4: Configure the pre-commit hook

**Files:**
- Modify: `.husky/pre-commit` (replace the default created by `husky init`)
- Modify: `package.json` (add `lint-staged` config section)

- [ ] **Step 1: Add lint-staged configuration to package.json**

Add this section to the root `package.json` (top-level, alongside `scripts` and `devDependencies`):

```json
"lint-staged": {
  "*.{ts,tsx}": [
    "biome check --write --no-errors-on-unmatched"
  ]
}
```

- [ ] **Step 2: Replace the pre-commit hook content**

Replace the content of `.husky/pre-commit` with:

```sh
pnpm lint-staged
turbo run typecheck test
```

This runs lint-staged first (Biome on staged files), then typecheck and test via Turbo.

- [ ] **Step 3: Verify lint-staged runs Biome on a staged file**

Create a temporary test by staging a file with a known lint issue, then running lint-staged manually:

```bash
pnpm lint-staged --verbose
```

Expected: lint-staged reports which files it processed and what commands it ran. If there are staged `.ts`/`.tsx` files, Biome runs on them.

- [ ] **Step 4: Commit the pre-commit hook and lint-staged config**

```bash
git add .husky/pre-commit package.json
git commit -m "chore(configs): add pre-commit hook with lint-staged, typecheck, and test"
```
