# `@repo/i18n`

Locale routing, dictionary loading, and translation utilities for the monorepo. Provides middleware-level locale detection, server-side dictionary loading, and a React context for client-side translations.

**npm name:** `@repo/i18n`  
**Path:** `packages/i18n`  
**Type:** Source-only (no build step)

---

## Overview

The i18n system has three layers:

1. **Routing** — `createI18nProxy` handles locale detection in Next.js middleware. The default locale (`en`) is hidden from URLs; non-default locales get a path prefix (`/ga/...`).
2. **Server-side loading** — `getDictionary` and `loadDictionary` load and merge categorized JSON dictionaries. Shared strings (from the package itself) are automatically merged with app-level strings.
3. **Client-side access** — `DictionaryProvider` + `useDictionary` thread the dictionary through React context for Client Components.

---

## Locale routing

### `createI18nProxy(config: I18nConfig)`

Factory returning a Next.js middleware proxy function. Call it once in `middleware.ts`.

```ts
import { createI18nProxy } from "@repo/i18n";

const proxy = createI18nProxy({
  defaultLocale: "en",
  locales: ["en", "ga"],
});

export function middleware(request: NextRequest) {
  return proxy(request);
}
```

**`I18nConfig`:**

| Field | Type | Description |
|---|---|---|
| `defaultLocale` | `string` | The default locale. Hidden from URLs — requests for `/en/...` are redirected to `/...`. |
| `locales` | `readonly string[]` | All supported locales, including the default. |

**URL routing behaviour:**

| Incoming request | Locale detection | Result |
|---|---|---|
| `/about/` (no prefix) | Accept-Language header → `en` (default) | Rewrite to `/{defaultLocale}/about/` internally |
| `/about/` (no prefix) | Accept-Language header → `ga` (non-default) | Redirect to `/ga/about/` |
| `/ga/about/` | Explicit `ga` prefix | `NextResponse.next()` — pass through |
| `/en/about/` | Explicit `en` (default) prefix | Redirect to `/about/` — strip default prefix |

Locale detection uses `Accept-Language` header negotiation via `@formatjs/intl-localematcher` and `negotiator`. If negotiation fails, falls back to `defaultLocale`.

## Dictionary format

Dictionaries are two-level categorized JSON files:

```json
{
  "nav": {
    "home": "Home",
    "about": "About us"
  },
  "errors": {
    "notFound": "Page not found",
    "items_one": "1 item",
    "items_other": "{count} items"
  }
}
```

Keys in the second level support dot notation for deeper nesting (`"social.fb": "Facebook"`). Plural forms use `_zero`, `_one`, `_two`, `_few`, `_many`, `_other` suffixes (CLDR plural categories).

The package ships its own shared dictionaries for `en` and `ga` (at `packages/i18n/dictionaries/`). App-level dictionaries are merged on top — app values win.

## Server-side loading

### `loadDictionary(locale, loaders, defaultLocale?)`

Loads and merges dictionaries for a locale. Returns a flat `Record<string, string>` (dotted keys).

```ts
async function loadDictionary(
  locale: string,
  loaders: DictionaryLoaders,
  defaultLocale?: string,
): Promise<Record<string, string>>
```

**Merge order (last wins):**
1. Default locale shared dictionary (from `@repo/i18n`)
2. Default locale app dictionary
3. Requested locale shared dictionary
4. Requested locale app dictionary

This means `ga` strings fall back to `en` for any keys not yet translated.

**`DictionaryLoaders`** — a record of async loader functions keyed by locale:

```ts
type DictionaryLoaders = Record<string, () => Promise<{ default: CategorizedDictionary }>>;
```

### `getDictionary<T>(locale, loaders, defaultLocale?)`

Loads, merges, and **unflattens** dictionaries for a locale. Returns a nested object where plural keys become `(count: number) => string` functions.

```ts
async function getDictionary<T = Record<string, unknown>>(
  locale: string,
  loaders: DictionaryLoaders,
  defaultLocale?: string,
): Promise<T>
```

This is the function to call in Server Components. Pass your app's `Dictionary` type as `T` for full type safety.

### `flattenCategorized(categorized)`

Converts a two-level categorized dictionary to a flat dotted-key record.

```ts
function flattenCategorized(
  categorized: Record<string, Record<string, string>>
): Record<string, string>
```

Example: `{ footer: { "social.fb": "FB" } }` → `{ "footer.social.fb": "FB" }`

Used internally by `loadDictionary`. You may need it when preparing the flat dictionary to pass to `DictionaryProvider`.

## React client-side access

### `DictionaryProvider`

A Client Component that provides the dictionary to its subtree via React context. Accepts the **flat** dictionary (serializable across the server-client boundary) and unflattens it on the client.

```tsx
"use client";
// In a layout or page component:

import { DictionaryProvider } from "@repo/i18n";

// Server Component loads the flat dictionary:
const flat = await loadDictionary(locale, loaders, defaultLocale);

// Pass it to the provider:
return (
  <DictionaryProvider flat={flat} locale={locale}>
    {children}
  </DictionaryProvider>
);
```

**Props:**

| Prop | Type | Description |
|---|---|---|
| `flat` | `Record<string, string>` | Flat dotted-key dictionary from `loadDictionary`. |
| `locale` | `string` | Active locale — used by `unflatten` to build plural functions. |
| `children` | `ReactNode` | Subtree that can call `useDictionary`. |

> Plural functions cannot cross the server-client serialization boundary. That is why `DictionaryProvider` receives the flat dictionary (plain strings) and calls `unflatten` on the client.

### `useDictionary<T>()`

Hook for accessing the dictionary in Client Components. Must be called within a `DictionaryProvider`.

```tsx
"use client";

import { useDictionary } from "@repo/i18n";
import type { Dictionary } from "@/types/dictionary"; // your app's Dictionary type

export function MyComponent() {
  const t = useDictionary<Dictionary>();
  return <span>{t.nav.home}</span>;
}
```

Throws if called outside a `DictionaryProvider`.

## String utilities

### `interpolate(template, values)`

Replaces `{key}` placeholders in a template string.

```ts
function interpolate(template: string, values: Record<string, string>): string
```

```ts
interpolate("Hello, {name}!", { name: "World" }); // "Hello, World!"
interpolate("Page {page} of {total}", { page: "2", total: "5" }); // "Page 2 of 5"
```

### `plural(group, count, locale)`

Selects the correct plural form for `count` using `Intl.PluralRules`, then interpolates `{count}`.

```ts
function plural(group: PluralGroup, count: number, locale: string): string
```

**`PluralGroup`** — partial record of CLDR plural categories:

```ts
type PluralGroup = Partial<Record<Intl.LDMLPluralRule, string>>;
// Keys: "zero" | "one" | "two" | "few" | "many" | "other"
```

```ts
plural({ one: "1 item", other: "{count} items" }, 1, "en"); // "1 item"
plural({ one: "1 item", other: "{count} items" }, 5, "en"); // "5 items"
// Explicit "zero" overrides CLDR rules (English maps 0 → "other"):
plural({ zero: "No items", one: "1 item", other: "{count} items" }, 0, "en"); // "No items"
```

In practice, you do not call `plural` directly. After `getDictionary`, plural keys in the dictionary are automatically converted to `(count: number) => string` functions by `unflatten`:

```ts
const dict = await getDictionary(locale, loaders);
dict.errors.items(5); // "5 items"
```

### `rich(template, tags)`

Parses `<tag>...</tag>` patterns in a template string and replaces them with React nodes.

```ts
function rich(template: string, tags: Record<string, RichTagFactory>): ReactNode[]
```

**`RichTagFactory`** — `(children: string) => ReactNode`

```ts
import { rich } from "@repo/i18n";

// Dictionary value: "Read our <link>privacy policy</link>"
const nodes = rich(t.footer.privacyText, {
  link: (text) => <a href="/privacy">{text}</a>,
});

return <p>{nodes}</p>;
```

Tags without a matching factory are passed through as plain text.

### `unflatten(flat, locale)`

Converts a flat dotted-key dictionary to a nested object. Plural-suffixed keys are grouped and converted to `(count: number) => string` functions.

```ts
function unflatten(flat: Record<string, string>, locale: string): Record<string, unknown>
```

Called internally by `getDictionary` and `DictionaryProvider`. You rarely need to call this directly.

## TypeScript types

### `MergedDictionary<TApp>`

Compile-time type utility that merges the shared package dictionary with your app dictionary, then produces the correct nested type with plural functions.

```ts
// packages/i18n/src/types.ts
import type app from "@/dictionaries/en.json";
export type Dictionary = MergedDictionary<typeof app>;
```

Use this as the type argument for `getDictionary<Dictionary>()` and `useDictionary<Dictionary>()` to get full autocompletion on dictionary keys.

### `I18nConfig`

Configuration shape for `createI18nProxy`:

```ts
type I18nConfig = {
  defaultLocale: string;
  locales: readonly string[];
};
```

### `CategorizedDictionary`

```ts
type CategorizedDictionary = Record<string, Record<string, string>>;
```

### `DictionaryLoaders`

```ts
type DictionaryLoaders = Record<string, () => Promise<{ default: CategorizedDictionary }>>;
```

## Complete server-side example

```ts
// app/[locale]/layout.tsx (Server Component)
import { getDictionary, loadDictionary, DictionaryProvider } from "@repo/i18n";

const loaders: DictionaryLoaders = {
  en: () => import("@/dictionaries/en.json"),
  ga: () => import("@/dictionaries/ga.json"),
};

export default async function LocaleLayout({ children, params }) {
  const { locale } = params;

  // For Server Components: full nested dictionary
  const dict = await getDictionary(locale, loaders, "en");

  // For Client Components: flat dictionary (serializable)
  const flat = await loadDictionary(locale, loaders, "en");

  return (
    <DictionaryProvider flat={flat} locale={locale}>
      <p>{dict.nav.home}</p>
      {children}
    </DictionaryProvider>
  );
}
```
